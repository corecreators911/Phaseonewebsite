# Coding Conventions

## TypeScript
- **Strict Mode**: Enabled in `tsconfig.json`.
- **Typing**: Use explicit types for component props and function return values.
- **Interfaces**: Prefer interfaces for data structures and prop definitions.

## Styling
- **Tailwind CSS 4**: Use modern Tailwind 4 syntax and CSS-first configuration.
- **Organization**: Group styles logically within component files.

## Components
- **Functional Components**: Use functional components with hooks exclusively.
- **State Management**: Prefer local state or context API for global state.
- **Naming**: PascalCase for component files and folders.

## Git & Workflow
- **Conventional Commits**: Use descriptive, standard commit messages.
- **GSD Integration**: All major work tracked via `.planning/` files.
